# Watchfire — User Use Cases

Things a user does or experiences. Three top-level cases, each broken
into action-shaped sub-cases. The engineering decomposition (and the
test acceptance criteria) lives in `behavior-spec.md`; tests are
written against those IDs but every test traces back to a UC here.

---

## UC-1 — Detailed window: every session at a glance

The expanded widget. Always-on-top, lists every running AI session.

- **1.1** — посмотреть на список и увидеть все активные чаты, сгруппированные по папке проекта
- **1.2** — у каждой сессии видеть % использованного контекста и лимит модели (`41% [1M]`)
- **1.3** — у каждой сессии видеть, сколько прошло с последней активности (`done · 2h`, `working · 23s`)
- **1.4** — если сессию не переименовывал, видеть на её месте начало первого промпта, а не UUID
- **1.5** — увидеть, какая сессия ждёт моего ответа (waiting → красная пульсирующая точка)
- **1.6** — кликнуть по сессии и переключиться на её терминал в Windows Terminal
- **1.7** — навести курсор на сессию и прочитать её последний промпт + модель
- **1.8** — открыть `[N]`-popup со всеми чатами папки (и активные, и старые)
- **1.9** — отличить Claude от Codex по виду строки (агент-бейдж + цвет крыши на iso-map)
- **1.10** — свернуть группу проекта, который сейчас не нужен

## UC-2 — Mini window: who's waiting on me, nothing else

Свёрнутая полоска для low-attention мониторинга.

- **2.1** — свернуть/развернуть виджет в мини-режим, и чтобы выбор запомнился между перезагрузками
- **2.2** — увидеть имена сессий, которые ждут моего ответа (не просто счётчик)
- **2.3** — увидеть, сколько сессий running и done (счётчики с нулём не показываются; `idle` не показывается совсем)
- **2.4** — кликнуть по ждущей сессии прямо из мини и переключиться на её терминал

## UC-3 — Hygiene under the hood

Поведение, которое юзер не делает руками, но ожидает что «оно само работает».

- **3.1** — после перезагрузки ноута не видеть вчерашние чаты
- **3.2** — после закрытия терминала перестать видеть его сессию (в течение ~5 минут)
- **3.3** — не видеть в виджете крон-задачи (daily-summary, fireflies-sync)
- **3.4** — изменения статуса прилетают в виджет в реальном времени, без F5
- **3.5** — после рестарта сервера виджет сам переподключается
- **3.6** — виджет остаётся поверх других окон (always-on-top)
- **3.7** — при запуске виджет сам подбирает высоту окна под текущее число сессий
- **3.8** — после `/rename` увидеть новое имя и в виджете, и на вкладке Windows Terminal
- **3.9** — одной командой `wf` запустить виджет / сервер / остановить / посмотреть статус
